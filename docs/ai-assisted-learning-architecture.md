## 1. 한 문장 정의

SSMK 투자 관찰노트는 AI가 미국주식 데이터를 수집하고, 초보자도 이해할 수 있는 가설과 해석을 만들고, 실제 시장 결과와 비교 복기하면서 생산자인 나부터 함께 배우는 학습형 투자 관찰 시스템이다.

이 프로젝트의 1차 소비자는 발행자인 나 자신이다. 따라서 이 시스템은 처음부터 나중에 이메일 발송까지 확장 가능한 구조를 염두에 두되, 핵심 목적은 종목을 찍어주는 것이 아니라 시장을 보는 눈을 훈련하는 것이다.

```text
AI = 여러 역할 에이전트로 나뉜 분석/작성/검증/감독 팀
나 = 첫 번째 학습자 + 질문자 + 편집장 + 발송 승인자
```

## 2. 핵심 철학

이 프로젝트는 다음 질문에서 출발한다.

```text
AI가 대신 판단해 주는 리포트를 만들 것인가?
아니면 AI와 함께 판단 근거를 배우는 리포트를 만들 것인가?
```

SSMK의 답은 두 번째다. AI가 분석과 가설을 자동으로 만들지만, 그 결과는 초보자인 내가 이해할 수 있어야 한다. 그래서 모든 핵심 가설은 반드시 아래 구조로 작성한다.

```text
가설 요약
→ 근거 지표/데이터
→ 이 지표/데이터를 보고 왜 그렇게 예상했는가
→ 초보자가 배울 포인트
→ 이 해석의 한계
→ 다음에 확인할 데이터
```

나쁜 예시는 다음과 같다.

```text
클라우드 성장률이 좋아서 MSFT는 긍정적으로 보입니다.
```

좋은 예시는 다음과 같다.

```text
가설 요약:
MSFT는 클라우드 매출 성장률이 유지된다면 AI 인프라 기대가 실제 실적으로 연결될 가능성이 있다.

근거 지표/데이터:
클라우드 매출 성장률, 영업이익률, AI 관련 설비투자 증가 여부.

왜 이렇게 해석했나:
클라우드 매출은 기업 고객이 실제로 돈을 내고 쓰는 서비스 매출이다.
AI 기대가 단순 뉴스가 아니라 실제 매출로 이어지려면 클라우드 성장률과 마진이 함께 유지되는지 봐야 한다.

초보자가 배울 포인트:
AI 테마는 뉴스만으로 판단하지 않고, 실제 고객이 돈을 쓰는 매출 지표와 회사가 남기는 이익률을 함께 확인해야 한다.

한계:
AI 기대가 이미 주가에 많이 반영되어 있을 수 있고, 설비투자 비용이 마진을 누를 수 있다.

다음 확인:
다음 실적 발표에서 Azure 성장률과 AI 관련 비용 설명을 확인한다.
```

## 3. 전체 흐름

```text
데이터 수집
  ↓
AI 데이터 정리
  ↓
벡터가 데이터 검증
  ↓
루미가 가설과 초보자용 해석 작성
  ↓
Weekly Lab 초안 생성
  ↓
세이지가 품질/추천화 위험 검토
  ↓
파일럿이 승인/발송/복기 상태 확인
  ↓
노바가 자동화 발전 제안 여부 판단
  ↓
사람이 학습자/편집장으로 읽고 질문
  ↓
승인 후 발송 또는 보관
  ↓
실제 시장 결과 추적
  ↓
과거 가설과 실제 결과 비교
  ↓
레슨&런 정리
  ↓
AI 품질 자가검증 / 발전 단계 제안
  ↓
사람 승인 또는 보류
  ↓
다음 리포트와 점수 모델 개선
```

여기서 중요한 점은 AI가 단순히 "좋다/나쁘다"를 말하는 것이 아니라, 왜 그렇게 생각했는지 초보자가 배울 수 있게 설명해야 한다는 것이다.

## 3-1. AI 리뷰 보드

SSMK에서는 AI를 하나의 역할로 보지 않고, 서로 견제하는 5개 핵심 에이전트로 나눈다.

| 이름 | 역할 | 주요 질문 |
|---|---|---|
| 루미 | 콘텐츠 크리에이터 | 초보자가 이해할 수 있는 리포트인가? |
| 벡터 | 데이터 검증 | 데이터가 최신이고 출처가 분명한가? |
| 세이지 | 편집자/품질 검증 | 추천처럼 읽히거나 과장된 표현은 없는가? |
| 파일럿 | 운영/프로세스 감독 | 승인, 발송, 복기 흐름이 끊기지 않았는가? |
| 노바 | 자동화 발전 제안 | 다음 단계로 넘어갈 근거가 충분한가? |

핵심 원칙:

```text
콘텐츠를 만든 에이전트는 자기 결과물을 최종 승인하지 않는다.
데이터 검증, 문장 검증, 운영 검증, 자동화 발전 판단을 분리한다.
```

자세한 역할과 충돌 해결 규칙은 `automation/ai-agent-roles-and-review-board.md`를 따른다.

## 4. AI의 역할

### 4-1. 데이터 수집 보조

AI는 다음 데이터를 모으거나 정리한다.

| 데이터 | 왜 보는가 |
|---|---|
| 주가 변화 | 시장이 해당 기업을 어떻게 평가했는지 보기 위해 |
| 52주 고점/저점 위치 | 현재 가격이 과열/침체 구간인지 참고하기 위해 |
| 매출 성장률 | 실제 돈벌이 규모가 커지는지 보기 위해 |
| EPS 성장률 | 주주 1주당 이익이 늘어나는지 보기 위해 |
| 영업이익률 | 돈을 벌 때 얼마나 남기는지 보기 위해 |
| 현금흐름 | 회계상 이익이 실제 현금으로 이어지는지 보기 위해 |
| 배당률/배당성향 | 주주환원이 지속 가능한지 보기 위해 |
| 실적 발표일 | 다음에 확인할 핵심 이벤트를 알기 위해 |
| 주요 뉴스 | 시장 기대와 리스크 변화를 추적하기 위해 |
| 산업 지표 | 기업이 속한 상권이 좋아지는지 보기 위해 |

### 4-2. 가설 생성

AI는 데이터를 보고 매주 가설을 만든다. 단, 가설은 예측 확정문이 아니라 학습용 관찰 문장이어야 한다.

가설은 아래 6단 구조를 따른다.

| 단계 | 질문 |
|---|---|
| 가설 요약 | 이번 데이터로 무엇을 예상해 볼 수 있는가? |
| 근거 지표/데이터 | 어떤 숫자나 이벤트를 봤는가? |
| 해석 이유 | 그 지표가 왜 그런 예상을 가능하게 하는가? |
| 초보자 레슨 | 이 지표가 시장이나 기업의 무엇을 보여주는가? |
| 한계 | 아직 무엇을 모르기 때문에 조심해야 하는가? |
| 다음 확인 | 이 가설이 맞는지 나중에 무엇으로 검증할 것인가? |

### 4-3. 초보자용 해석

AI는 모든 어려운 지표를 쉬운 비유와 함께 설명한다.

예시:

```text
영업이익률은 매출에서 비용을 빼고 얼마나 남겼는지 보는 지표입니다.
카페로 비유하면, 커피를 1만 원어치 팔았을 때 재료비, 인건비, 임대료를 빼고 얼마가 남는지 보는 것입니다.
매출이 늘어도 영업이익률이 떨어지면, 많이 팔았지만 남는 돈은 줄어드는 상황일 수 있습니다.
```

### 4-4. 솔직한 한계 고지

AI는 자신이 모르는 것을 숨기지 않는다.

좋은 한계 고지:

```text
이번 가설의 한계:
이 분석은 최신 실적 발표 전 데이터를 기준으로 작성되었습니다.
뉴스는 긍정적으로 보이지만, 실제 매출과 이익에 반영되었는지는 아직 확인되지 않았습니다.
따라서 이번 판단은 확정이 아니라 다음 실적에서 검증할 관찰 가설입니다.
```

### 4-5. 복기와 개선

AI는 과거 리포트와 실제 시장 결과를 비교한다.

복기 질문:

- 지난 가설은 실제 주가/실적/뉴스 흐름과 얼마나 맞았는가?
- 맞았다면 어떤 지표가 도움이 되었는가?
- 틀렸다면 어떤 지표를 과대평가했는가?
- 초보자인 내가 이해하기 쉽게 설명되었는가?
- 다음 리포트에서 설명 방식이나 점수 기준을 어떻게 바꿀 것인가?

## 5. 사람의 역할

사람은 투자 전문가 심사자가 아니다. 사람은 첫 번째 학습자이자 편집장이다.

사람이 하는 일:

1. AI 리포트를 읽는다.
2. 이해가 안 되는 지표에 질문한다.
3. 설명이 너무 어려운 부분을 표시한다.
4. 이메일로 보내도 되는 표현인지 확인한다.
5. 발송 여부를 승인한다.
6. 다음 리포트에서 더 알고 싶은 질문을 남긴다.
7. AI가 제안한 자동화 발전 단계나 중요한 변경을 승인/보류/거절한다.

사람이 꼭 전문가처럼 해야 하는 일은 아니다.

```text
내가 이 분석이 맞는지 판단한다
```

보다는 아래가 맞다.

```text
내가 이 분석을 이해했는지 확인하고, 이해가 안 되는 부분을 AI에게 다시 묻는다.
```

## 6. 리포트 구조

주간 리포트는 나중에 이메일 발송 단계로도 확장할 수 있도록 구조를 잡되, 현재는 학습용 Weekly Lab 초안 기준으로 작성한다.

권장 구조:

1. 이번 주 3줄 요약
2. 시장 온도계
3. 이번 주 산업 관찰 우선순위 3개
4. SSMK 관찰 우선순위 3개
5. 이번 주 AI 가설 3개
6. 가설별 근거 지표와 해석 이유
7. AI의 솔직한 한계
8. 밸류에이션 재점검 후보 3개
9. 리스크 먼저 확인할 후보 3개
10. 지난 가설과 실제 결과 비교
11. 이번 주 레슨&런
12. 다음 주 확인할 질문
13. 데이터 출처와 면책 문구

핵심은 5번과 6번이다. 초보자가 리포트를 읽고 이런 생각을 해야 한다.

```text
아, 매출 성장률은 회사의 덩치가 커지는지를 보여주는구나.
아, 영업이익률은 많이 팔아도 남는 돈이 줄어들 수 있다는 걸 보여주는구나.
아, 배당률 상승은 좋은 신호일 수도 있지만 주가 하락 착시일 수도 있구나.
```

## 8. 자동화 원칙

이 프로젝트는 기술적으로 가능한 범위에서 최대한 자동화한다.

자동화할 것:

- 데이터 수집
- 지표 계산
- 점수 계산
- 관찰 등급 변환
- 데이터 신뢰도 초안 판단
- AI 가설 생성
- 초보자용 해석 작성
- 리포트 초안 작성
- 과거 가설과 실제 결과 비교 초안
- 레슨&런 초안
- 에이전트별 품질 리뷰 초안

초기에도 사람에게 남길 것:

- 이메일 발송 승인
- 이해가 안 되는 설명에 대한 질문
- 외부 공유 전에 민감한 표현 확인
- 자동화 발전 단계 승인
- 점수 모델, 데이터 출처, 프롬프트 구조처럼 중요한 변경 승인

즉, 목표는 수동 작성이 아니라 자동 생성이다. 다만 발송 버튼은 처음부터 완전 자동으로 열지 않는다.
또한 AI가 품질이 안정됐다고 판단해도, 다음 자동화 단계는 사용자 승인 없이 적용하지 않는다.

## 9. 발전 승인 게이트

AI는 주간 리포트와 월간 리뷰가 쌓일수록 콘텐츠 품질을 자가 검증한다.

검증할 항목:

- AI 가설이 6단 구조를 지키는가?
- 근거 지표와 해석 이유가 실제로 연결되는가?
- 초보자가 지표 의미를 배울 수 있는가?
- 데이터 신뢰도와 불확실성을 솔직하게 표시하는가?
- 지난 가설과 실제 결과 비교가 가능한가?
- 사용자가 반복해서 수정하거나 질문하는 부분이 줄어드는가?

AI가 다음 단계로 넘어가도 된다고 판단하면 바로 적용하지 않고, 아래 형식으로 사용자에게 제안한다.

```text
현재 상태
AI가 그렇게 판단한 근거
추천하는 발전 단계
기대 효과
위험과 한계
되돌리는 방법
사용자 승인 요청
```

예시:

```text
최근 4회 리포트에서 AI 가설 6단 구조가 안정적으로 유지됐고,
사용자 수정도 문장 표현 중심으로 줄었습니다.
이제 주간 리포트 초안 생성 자동화를 강화해도 될 단계로 보입니다.
다만 자동 생성 문장이 투자 추천처럼 보일 위험은 계속 있으므로,
발송 승인은 그대로 유지하는 안을 추천합니다.
이 방향으로 진행해도 될까요?
```

자세한 기준은 `automation/automation-evolution-approval-gate.md`를 따른다.

## 10. 발전 단계

### 1단계: AI 자동 초안 + 사람 학습 승인

- Google Sheets에 기본 데이터 저장
- AI가 리포트 초안 생성
- 사람은 읽고 질문한 뒤 발송 승인

### 2단계: 가설 추적 자동화

- 가설별 `hypothesis_id` 부여
- 1주/4주 후 실제 결과 자동 비교
- 맞았는지보다 왜 그렇게 움직였는지 복기

### 3단계: 점수 모델 개선 자동 제안

- AI가 지난 복기에서 반복된 오류를 찾음
- 예: 뉴스 기대를 너무 크게 반영함, 밸류에이션 부담을 낮게 봄
- 다음 달 점수 모델 개선안을 제안

### 4단계: 이메일/PDF/아카이브 자동화 (장기 목표)

- 승인된 리포트만 Gmail 발송
- PDF 생성
- Drive와 로컬 reports 폴더에 보관
- Notion에 레슨&런 요약 저장

중요:

```text
각 단계로 넘어갈 때는 AI가 승급 제안을 만들고, 사용자가 승인한 뒤 적용한다.
```
