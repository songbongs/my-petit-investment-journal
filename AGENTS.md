# AGENTS.md

이 프로젝트에서 작업하는 모든 Codex/AI/사람 유지보수자는 먼저 아래 문서를 읽는다.

- `docs/SSMK-WEEKLY-LAB-SOUL.md`
- `docs/2026-04-30-external-project-benchmark-notes.md` (외부 프로젝트 벤치마킹이 필요한 개발일 때)

이 문서는 SSMK Weekly Lab의 최우선 방향성 문서다. 보고서, 자동화, Google Sheets, Apps Script, 이메일 HTML, 대시보드, 프롬프트를 수정할 때 이 문서의 원칙을 우선한다.

핵심 기준:

1. SSMK Weekly Lab은 투자 추천이 아니라 투자 공부용 관찰 시스템이다.
2. 최종 콘텐츠는 `실제 변화 -> 해석 -> 초보자 레슨 -> 다음 확인 질문` 순서로 작성한다.
3. Docs 초안, 이메일 HTML, 운영 대시보드의 역할을 섞지 않는다.
4. Dividend & ETF Corner, 산업/테마 보드, 핵심 가설, Forecast vs Actual, 가설 진화 흐름을 누락하지 않는다.
5. 데이터가 부족하면 사실처럼 쓰지 말고 무엇이 부족한지와 왜 필요한지를 쉬운 말로 설명한다.
6. 이메일 자동 발송은 기본 OFF이며, 발송은 사용자 승인 후 진행한다.
7. 모든 설명은 비개발자/초보 사용자가 이해할 수 있게 한국어로 쉽게 쓴다.

작업 전 질문:

```text
이 변경이 사용자가 더 좋은 투자 공부 질문을 하게 도와주는가?
```

대답이 아니면 방향을 다시 잡는다.
